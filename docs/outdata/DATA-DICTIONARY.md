# Output Data ‚Äî Data Dictionary

**Directory**: `outdata/`
**Dictionary type**: Output / processed data
**Last updated**: 2026-03-28
**Maintained by**: Jason Pither

Processed, analysis-ready CSV files produced by the wrangling scripts in `scripts/`.
All files are in long (tidy) format with one observation per row. These files are
read directly by `index.Rmd` to produce the analysis website.

Each wrangling script implements a **checkpoint**: if the output file already exists
and is newer than all contributing raw data files, the script skips processing and
prints a "up-to-date" message. Delete the relevant output file to force regeneration.

### solar_data.csv

**Path**: `outdata/solar_data.csv`
**Generated by**: `scripts/wrangle_solar_data.R`
**Regenerate**: Delete this file, or add new `.xls` files to `rawdata/solar_data/` and re-render `index.Rmd`
**Git-ignored**: No

Hourly solar energy production in long format, consolidated from all AP Systems weekly
`.xls` reports in `rawdata/solar_data/`. Duplicate rows (from any overlapping weekly
downloads) are removed via `distinct()`. Rows with `kWh > 30` represent periods when
the AP Systems datalogger was taken offline and the system accumulated totals across
multiple hours; these anomalous values are flagged in `index.Rmd`.

#### Column schema

| Column | Type | Units | Description | Nullable |
|---|---|---|---|---|
| `date` | datetime | ‚Äî | Full timestamp: date + hour (`YYYY-MM-DD HH:00:00`) | No |
| `day` | date | ‚Äî | Calendar date (`YYYY-MM-DD`) | No |
| `hour` | integer | 24-hour clock | Hour of the day (0‚Äì23) | No |
| `kWh` | numeric | kWh | Solar energy produced by the array during this hour; 0 at night | Yes |

**Coverage**: July 8, 2022 ‚Äì present.
**Grain**: One row per (day, hour) combination; 24 rows per day.

### fortis_data.csv

**Path**: `outdata/fortis_data.csv`
**Generated by**: `scripts/wrangle_electricity_data.R`
**Regenerate**: Replace the file in `rawdata/fortis_data/` with a newer Fortis export and re-render `index.Rmd`
**Git-ignored**: No

Hourly electricity delivery and return data from the Fortis BC smart meter, covering
June 2022 to present. In `index.Rmd`, this file is bound with `fortis_historical_data.csv`
(via `dplyr::bind_rows`) and the combined dataset's columns are renamed:
`kwh_returned` ‚Üí `kw_solar_to_grid`, `kwh_consumed` ‚Üí `kw_fortis_from_grid`.

#### Column schema

| Column | Type | Units | Description | Nullable |
|---|---|---|---|---|
| `day` | date | ‚Äî | Calendar date (`YYYY-MM-DD`) | No |
| `hour` | integer | 24-hour clock | Hour of the day (0‚Äì23) | No |
| `kwh_returned` | numeric | kWh | Energy exported to the grid from the solar array during this hour | Yes |
| `kwh_consumed` | numeric | kWh | Energy drawn from the grid by the household during this hour | Yes |

**Coverage**: June 1, 2022 ‚Äì present.
**Grain**: One row per (day, hour); 24 rows per day.

### fortis_historical_data.csv

**Path**: `outdata/fortis_historical_data.csv`
**Generated by**: `scripts/wrangle_historical_electricity_data.R`
**Regenerate**: This file is derived from a static historical export; re-run only if the raw source file changes
**Git-ignored**: No

Hourly electricity consumption from before the solar installation. Column structure is
identical to `fortis_data.csv` so the two files can be combined with `dplyr::bind_rows`.
`kwh_returned` is `NA` for all rows (no solar-to-grid data exists for this period).

#### Column schema

| Column | Type | Units | Description | Nullable |
|---|---|---|---|---|
| `day` | date | ‚Äî | Calendar date (`YYYY-MM-DD`) | No |
| `hour` | integer | 24-hour clock | Hour of the day (0‚Äì23) | No |
| `kwh_returned` | numeric | ‚Äî | Always `NA`; no solar return data for the pre-installation period | Yes (always) |
| `kwh_consumed` | numeric | kWh | Energy drawn from the grid during this hour | Yes |

**Coverage**: January 1, 2019 ‚Äì May 31, 2022.
**Grain**: One row per (day, hour); 24 rows per day.

### weather_data.csv

**Path**: `outdata/weather_data.csv`
**Generated by**: `scripts/weather.R` (run manually with `force_update <- TRUE`)
**Regenerate**: Set `force_update <- TRUE` in `scripts/weather.R` and source the script
**Git-ignored**: No

Hourly weather observations from the UBCO Okanagan weather station (the closest
Environment Canada station to the solar installation). Downloaded via the `weathercan`
package. Currently covers July 8, 2022 through the last download date. No publicly
available solar radiation data exists for Kelowna; this file is retained for potential
future use in correlation analyses.

#### Key columns (selected from weathercan output)

The `weathercan` package returns a large number of columns. Key fields are:

| Column | Type | Units | Description | Nullable |
|---|---|---|---|---|
| `station_id` | integer | ‚Äî | Environment Canada station identifier | No |
| `station_name` | character | ‚Äî | Station name (e.g., `"KELOWNA UBCO"`) | No |
| `time` | datetime | ‚Äî | Observation timestamp (UTC) | No |
| `temp` | numeric | ¬∞C | Hourly air temperature | Yes |
| `rel_hum` | numeric | % | Relative humidity | Yes |
| `wind_spd` | numeric | km/h | Wind speed | Yes |
| `weather` | character | ‚Äî | Weather condition description (e.g., `"Clear"`, `"Cloudy"`) | Yes |

See the [weathercan documentation](https://docs.ropensci.org/weathercan/) for the
full column list.

**Coverage**: July 8, 2022 ‚Äì last manual download date.
